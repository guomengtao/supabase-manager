Supabase API Keys Guide

Understanding and using different types of API keys in your Supabase project.

Public (anon) Key
Use Cases:
  • Client-side applications
  • Public-facing websites
  • Mobile apps
  • Browser applications
Characteristics:
  • Respects Row Level Security (RLS)
  • Safe to expose in client code
  • Limited to user-specific data
  • Requires proper authentication
Service Role Key
üîí Critical Security: This key bypasses Row Level Security. Never share it publicly!
Use Cases:
  • Server-side applications
  • Administrative tasks
  • Data migrations
  • Secure background jobs
Characteristics:
  • Bypasses Row Level Security
  • Full database access
  • Must be kept secure
  • Server-side only
Best Practices
When to Use Public (anon) Key:
  • In browser-based applications
  • Mobile applications
  • Public-facing websites
  • When user authentication is handled by Supabase Auth
When to Use Service Role Key:
  • Server-side applications (Node.js, Python, etc.)
  • Administrative scripts
  • Data migrations and backups
  • When bypassing RLS is necessary
